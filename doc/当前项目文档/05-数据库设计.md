## 3. 数据表详细设计

### 3.1 admin (管理员表)

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | INT | 10 | 否 | AUTO_INCREMENT | 主键 |
| name | VARCHAR | 45 | 否 | - | 管理员用户名 |
| password | VARCHAR | 45 | 否 | - | 密码 |

**索引**:
- PRIMARY KEY (id)
- UNIQUE KEY name_UNIQUE (name)

### 3.2 consumer (用户表)

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | INT | 10 UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| username | VARCHAR | 255 | 否 | - | 用户名 |
| password | VARCHAR | 100 | 否 | - | 密码 |
| sex | TINYINT | 4 | 是 | NULL | 性别 (0:女, 1:男) |
| phone_num | CHAR | 15 | 是 | NULL | 手机号 |
| email | CHAR | 30 | 是 | NULL | 邮箱 |
| birth | DATETIME | - | 是 | NULL | 生日 |
| introduction | VARCHAR | 255 | 是 | NULL | 个人简介 |
| location | VARCHAR | 45 | 是 | NULL | 所在地 |
| avator | VARCHAR | 255 | 是 | NULL | 头像URL |
| create_time | DATETIME | - | 否 | - | 创建时间 |
| update_time | DATETIME | - | 否 | - | 更新时间 |

**索引**:
- PRIMARY KEY (id)
- UNIQUE KEY username_UNIQUE (username)
- UNIQUE KEY phone_num_UNIQUE (phone_num)
- UNIQUE KEY email_UNIQUE (email)

### 3.3 singer (歌手表)

| 字段名 | 类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|--------|------|
| id | INT | 10 UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| name | VARCHAR | 45 | 否 | - | 歌手名 |

## 4. 表关系说明

### 4.1 一对多关系

| 主表 | 从表 | 关系说明 |
|------|------|----------|
| singer | song | 一个歌手有多首歌曲 |
| consumer | collect | 一个用户有多个收藏 |
| consumer | comment | 一个用户有多条评论 |
| song | comment | 一首歌曲有多条评论 |
| song_list | comment | 一个歌单有多条评论 |
| song_list | list_song | 一个歌单有多首歌曲 |
| song | list_song | 一首歌曲可以在多个歌单中 |

### 4.2 多对多关系

| 表1 | 表2 | 关联表 | 关系说明 |
|-----|-----|--------|----------|
| song | song_list | list_song | 歌曲与歌单的多对多关系 |
| consumer | comment | user_support | 用户与评论的点赞关系 |

## 5. 关键业务逻辑

### 5.1 收藏逻辑

- type 字段区分收藏类型：
  - 0: 收藏歌曲 (song_id 有值, song_list_id 为 NULL)
  - 1: 收藏歌单 (song_list_id 有值, song_id 为 NULL)

### 5.2 评论逻辑

- type 字段区分评论类型：
  - 0: 歌曲评论 (song_id 有值, song_list_id 为 NULL)
  - 1: 歌单评论 (song_list_id 有值, song_id 为 NULL)
- up 字段记录点赞数

### 5.3 评分逻辑

- 用户对歌单的评分唯一（通过 unique 约束保证）
- score 范围：1-10

### 5.4 点赞逻辑

- user_support 表记录用户对评论的点赞
- 防止重复点赞（通过 comment_id + user_id 组合判断）

## 6. 数据库初始化

### 6.1 创建数据库

```sql
CREATE DATABASE tp_music 
CHARACTER SET utf8 
COLLATE utf8_general_ci;
```

### 6.2 导入数据

```bash
mysql -u root -p tp_music < tp_music.sql
```

## 7. 性能优化建议

### 7.1 索引优化

- 所有外键字段已建立索引
- 查询频繁的字段建议添加索引：
  - song.name (歌曲搜索)
  - song_list.title (歌单搜索)
  - singer.name (歌手搜索)

### 7.2 分页查询

- 列表查询建议使用分页
- 使用 LIMIT 和 OFFSET 进行分页

### 7.3 缓存策略

- 热点数据使用 Redis 缓存
- 验证码使用 Redis 存储（5分钟过期）
