# 后端架构文档 - music-server

## 1. 项目概述

music-server 是音乐网站的后端服务，基于 Spring Boot 框架开发，提供 RESTful API 接口供前端调用。

## 2. 技术架构

| EasyExcel | 3.1.3 | Excel 处理 |

## 3. 分层架构

### 3.1 Controller 层

负责接收 HTTP 请求，调用 Service 层，返回统一响应。

#### 主要控制器

| 控制器 | 功能 | API 前缀 |
|--------|------|----------|
| AdminController | 管理员登录 | /admin |
| ConsumerController | 用户管理 | /user |
| SingerController | 歌手管理 | /singer |
| SongController | 歌曲管理 | /song |
| SongListController | 歌单管理 | /songList |
| ListSongController | 歌单歌曲管理 | /listSong |
| CollectController | 收藏管理 | /collection |
| CommentController | 评论管理 | /comment |
| RankListController | 评分管理 | /rankList |
| BannerController | 轮播图管理 | /banner |
| UserSupportController | 点赞管理 | /userSupport |
| FileDownloadController | 文件下载 | - |
| MinioController | MinIO操作 | - |
| MinioUploadController | MinIO上传 | - |

### 3.2 Service 层

负责业务逻辑处理，调用 Mapper 层进行数据操作。

#### 服务接口与实现

| 服务接口 | 实现类 | 主要功能 |
|----------|--------|----------|
| AdminService | AdminServiceImpl | 管理员登录验证 |
| ConsumerService | ConsumerServiceImpl | 用户增删改查、登录 |
| SingerService | SingerServiceImpl | 歌手增删改查 |
| SongService | SongServiceImpl | 歌曲增删改查、文件上传 |
| SongListService | SongListServiceImpl | 歌单增删改查 |
| ListSongService | ListSongServiceImpl | 歌单歌曲关联管理 |
| CollectService | CollectServiceImpl | 收藏增删查 |
| CommentService | CommentServiceImpl | 评论增删查、点赞 |
| RankListService | RankListServiceImpl | 评分提交、查询 |
| BannerService | BannerServiceImpl | 轮播图查询 |
| UserSupportService | UserSupportServiceImpl | 点赞记录管理 |
| OrderManager | SimpleOrderManager | 订单/验证码管理 |

### 3.3 Mapper 层

负责数据库访问，使用 MyBatis Plus 进行 CRUD 操作。

### 3.4 Model 层

#### Domain 实体类

| 实体类 | 对应表 | 说明 |
|--------|--------|------|
| Admin | admin | 管理员 |
| Consumer | consumer | 用户 |
| Singer | singer | 歌手 |
| Song | song | 歌曲 |
| SongList | song_list | 歌单 |
| ListSong | list_song | 歌单歌曲关联 |
| Collect | collect | 收藏 |
| Comment | comment | 评论 |
| RankList | rank_list | 评分 |
| Banner | banner | 轮播图 |
| UserSupport | user_support | 用户点赞 |

#### Request DTO

用于接收前端请求参数，与 Domain 分离，便于参数校验和业务处理。

## 4. 统一响应封装

### 4.1 R 类

```java
public class R {
    private int code;       // 状态码
    private String msg;     // 消息
    private Object data;    // 数据
    private boolean success; // 是否成功
    
    // 静态方法
    public static R success(String msg)           // 成功响应
    public static R success(String msg, Object data)  // 成功带数据
    public static R error(String msg)             // 错误响应
    public static R fatal(String msg)             // 严重错误
    public static R warning(String msg)           // 警告
}
```

### 4.2 响应码定义

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求错误 |
| 401 | 未授权 |
| 500 | 服务器错误 |

## 7. 跨域配置

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins("*")
                .allowedMethods("*")
                .allowedHeaders("*")
                .allowCredentials(true);
    }

```

## 8. 文件上传配置

### 8.1 文件大小限制

```java
@Bean
public MultipartConfigElement multipartConfigElement() {
    MultipartConfigFactory factory = new MultipartConfigFactory();
    factory.setMaxFileSize(DataSize.of(20, DataUnit.MEGABYTES));      // 单个文件最大20M
    factory.setMaxRequestSize(DataSize.of(20, DataUnit.MEGABYTES));   // 总请求最大20M
    return factory.createMultipartConfig();
}
```

### 8.2 支持的文件类型

- 音频文件：MP3
- 图片文件：JPG, PNG
- 歌词文件：LRC
