# 部署指南

## 1. 环境准备

### 1.1 系统要求

- **操作系统**: Linux / macOS / Windows
- **内存**: 至少 4GB RAM
- **磁盘**: 至少 10GB 可用空间

### 1.2 软件依赖

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 1.8+ | Java 开发环境 |
| MySQL | 5.7+ | 数据库 |
| Redis | 5.0.8+ | 缓存 |
| Node.js | 14.17.3+ | 前端运行环境 |
| Maven | 3.6+ | Java 构建工具 |
| Nginx | 1.18+ | Web 服务器（可选） |

## 2. 本地开发环境部署

### 2.1 数据库部署

#### 2.1.1 安装 MySQL

```bash
# macOS (使用 Homebrew)
brew install mysql@5.7
brew services start mysql@5.7

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install mysql-server-5.7
sudo service mysql start

# Windows
# 下载安装包并安装
```

#### 2.1.2 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE tp_music CHARACTER SET utf8 COLLATE utf8_general_ci;

# 导入数据
mysql -u root -p tp_music < music-server/sql/tp_music.sql
```

#### 2.1.3 配置数据库用户

```sql
-- 创建用户（可选）
CREATE USER 'music_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON tp_music.* TO 'music_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2.2 Redis 部署

#### 2.2.1 安装 Redis

```bash
# macOS
brew install redis
brew services start redis

# Ubuntu/Debian
sudo apt-get install redis-server
sudo service redis-server start

# Windows
# 下载 Redis for Windows 并安装
```

#### 2.2.2 验证 Redis

```bash
redis-cli ping
# 应返回 PONG
```

### 2.3 后端服务部署

#### 2.3.1 配置数据库连接

编辑 `music-server/src/main/resources/application-dev.properties`：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/tp_music?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=your_password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

#### 2.3.2 配置邮件服务（可选，用于密码重置）

```properties
spring.mail.host=smtp.qq.com
spring.mail.port=587
spring.mail.username=your_email@qq.com
spring.mail.password=your_auth_code
spring.mail.protocol=smtp
spring.mail.default-encoding=UTF-8
```

#### 2.3.3 启动服务

```bash
cd music-server

# 使用 Maven
mvn spring-boot:run

# 或使用 Maven Wrapper
./mvnw spring-boot:run
```

服务启动后，访问 `http://localhost:8888` 验证。

### 2.4 前端项目部署

#### 2.4.1 安装依赖

```bash
# 前台客户端
cd music-client
npm install

# 后台管理端
cd music-manage
npm install
```

#### 2.4.2 配置 API 地址

编辑 `music-client/src/api/request.ts` 和 `music-manage/src/api/request.ts`：

```typescript
const baseUrl = 'http://localhost:8888';
```

#### 2.4.3 启动开发服务器

```bash
# 前台客户端
cd music-client
npm run serve
# 访问 http://localhost:8080

# 后台管理端
cd music-manage
npm run serve
# 访问 http://localhost:8081
```

## 3. 生产环境部署

### 3.1 后端打包部署

#### 3.1.1 打包

```bash
cd music-server
mvn clean package -DskipTests
```

生成的 jar 包位于 `target/yin-0.0.1-SNAPSHOT.jar`

#### 3.1.2 运行

```bash
# 直接运行
java -jar target/yin-0.0.1-SNAPSHOT.jar

# 后台运行
nohup java -jar target/yin-0.0.1-SNAPSHOT.jar > app.log 2>&1 &

# 指定配置文件运行
java -jar target/yin-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

### 3.2 前端打包部署

#### 3.2.1 前台客户端打包

```bash
cd music-client
npm run build
```

生成的静态文件位于 `dist/` 目录。

#### 3.2.2 后台管理端打包

```bash
cd music-manage
npm run build
```

生成的静态文件位于 `dist/` 目录。

#### 3.2.3 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前台客户端
    location / {
        root /path/to/music-client/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后台管理端
    location /admin {
        alias /path/to/music-manage/dist;
        index index.html;
        try_files $uri $uri/ /admin/index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3.3 Docker 部署

#### 3.3.1 服务端 Dockerfile

```dockerfile
# music-server/docker-server/dockerfile
FROM openjdk:8-jdk-alpine

WORKDIR /app

COPY target/yin-0.0.1-SNAPSHOT.jar app.jar

EXPOSE 8888

ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 3.3.2 Docker Compose 配置

```yaml
# docker-compose.yml
version: '3'

services:
  mysql:
    image: mysql:5.7
    container_name: music-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: tp_music
    volumes:
      - ./music-server/sql/tp_music.sql:/docker-entrypoint-initdb.d/init.sql
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  redis:
    image: redis:5.0
    container_name: music-redis
    ports:
      - "6379:6379"

  server:
    build: ./music-server
    container_name: music-server
    depends_on:
      - mysql
      - redis
    ports:
      - "8888:8888"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/tp_music?useUnicode=true&characterEncoding=utf-8&useSSL=false
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: 123456
      SPRING_REDIS_HOST: redis

  client:
    build: ./music-client
    container_name: music-client
    ports:
      - "8080:80"

  manage:
    build: ./music-manage
    container_name: music-manage
    ports:
      - "8081:80"

volumes:
  mysql_data:
```

#### 3.3.3 构建并运行

```bash
# 构建并启动所有服务
docker-compose up --build

# 后台运行
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 5. 安全配置

### 5.1 数据库安全

- 使用强密码
- 限制数据库访问 IP
- 定期备份数据

### 5.2 应用安全

- 启用 HTTPS
- 配置 CORS 白名单
- 设置合理的 Session 过期时间

### 5.3 服务器安全

- 配置防火墙
- 禁用 root 远程登录
- 定期更新系统补丁

## 7. 常见问题

### 7.1 数据库连接失败

**问题**: 无法连接 MySQL 数据库

**解决**:
1. 检查 MySQL 服务是否启动
2. 检查数据库配置是否正确
3. 检查防火墙设置

### 7.2 图片/音乐加载失败

**问题**: 前端无法加载图片或音乐文件

**解决**:
1. 检查资源文件是否存在
2. 检查资源路径配置
3. 确保资源目录有读取权限

### 7.3 跨域问题

**问题**: 前端请求报跨域错误

**解决**:
1. 检查后端 CORS 配置
2. 确保 `allowedOrigins` 包含前端域名

### 7.4 Redis 连接失败

**问题**: 无法连接 Redis

**解决**:
1. 检查 Redis 服务是否启动
2. 检查 Redis 配置是否正确
3. 检查防火墙设置

## 9. 备份与恢复

### 9.1 数据库备份

```bash
# 备份数据库
mysqldump -u root -p tp_music > backup_$(date +%Y%m%d_%H%M%S).sql

# 自动备份脚本（添加到 crontab）
0 2 * * * mysqldump -u root -p'password' tp_music > /backup/music_$(date +\%Y\%m\%d).sql
```

### 9.2 数据库恢复

```bash
# 恢复数据库
mysql -u root -p tp_music < backup_20240101.sql
```

### 9.3 文件备份

```bash
# 备份资源文件
tar -czvf music_resources_$(date +%Y%m%d).tar.gz music-server/img music-server/song
```

## 10. 更新部署

### 10.1 后端更新

```bash
# 1. 备份当前版本
cp target/yin-0.0.1-SNAPSHOT.jar backup/yin-$(date +%Y%m%d).jar

# 2. 拉取最新代码
git pull

# 3. 重新打包
mvn clean package -DskipTests

# 4. 重启服务
# 先停止旧服务
pkill -f yin-0.0.1-SNAPSHOT.jar

# 启动新服务
nohup java -jar target/yin-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

### 10.2 前端更新

```bash
# 1. 拉取最新代码
git pull

# 2. 安装依赖
npm install

# 3. 重新打包
npm run build

# 4. 更新 Nginx 静态文件
rm -rf /usr/share/nginx/html/*
cp -r dist/* /usr/share/nginx/html/

# 5. 重启 Nginx
nginx -s reload
```
