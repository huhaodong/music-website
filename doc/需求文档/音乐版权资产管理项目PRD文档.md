# 音乐版权资产管理库项目PRD文档

## 1. 产品概述

### 1.1 项目背景

本项目是为版权代理公司设计的音乐版权资产管理库，旨在集中管理公司产出的所有音乐相关物料内容，包括词、曲、谱子、合同文档等。通过本系统，用户可以高效地管理、搜索、预览和分享音乐版权资产，同时实现精细化的权限控制和数据可视化分析。

### 1.2 项目目标

- 实现音乐版权资产的集中管理和快速检索
- 提供音乐预览和试听功能，支持歌词同步显示
- 建立完整的鉴权系统，实现精细化的权限控制
- 支持物料的批量上传和管理
- 提供数据看板，实现版权资产数据的可视化分析
- 预留微信小程序扩展能力，支持外部用户提交发歌需求
- 实现网站的定制化管理

### 1.3 目标用户

- **内部员工**：公司内部的音乐制作、版权管理等相关人员
- **外部合作伙伴**：如词曲作者、制作人、发行平台等外部合作方
- **普通用户**：通过小程序等入口提交发歌需求的外部用户
- **其他用户**：其他与音乐版权相关的用户

### 1.4 产品价值

- **提高管理效率**：集中管理音乐版权资产，减少分散管理的复杂性
- **提升检索速度**：通过标签和搜索系统，快速找到所需的音乐物料
- **增强权限控制**：精细化的权限管理，确保敏感资产的安全
- **优化用户体验**：响应式设计，在不同设备上提供良好的使用体验
- **数据驱动决策**：通过看板系统，直观了解版权资产的分布和趋势

## 3. 核心流程

### 3.1 用户登录流程

1. 用户访问系统登录页面
2. 输入账号和密码
3. 系统验证用户身份
4. 验证成功后，根据用户角色和权限跳转到相应的首页
5. 验证失败时，显示错误提示

### 3.2 歌曲项目管理流程

1. 用户登录系统
2. 进入项目管理页面
3. 点击"创建项目"按钮
4. 填写项目基本信息（歌曲名称、艺术家、创作时间等）
5. 上传相关物料（音频、歌词、谱子、合同等）
6. 设置项目状态（已发行、未发行、暂缓发行）
7. 设置项目权限和标签
8. 保存项目信息
9. 系统生成项目记录，并关联相关物料

### 3.3 物料上传流程

1. 用户进入物料管理页面或项目详情页面
2. 点击"上传物料"按钮
3. 选择文件（支持批量选择、拖拽上传、文件夹上传）
4. 填写物料信息（文件名称、类型、描述、标签等）
5. 设置物料权限
6. 点击"上传"按钮
7. 系统上传文件并生成预览
8. 上传完成后，系统自动分类和标签，并通知相关人员

### 3.4 音乐试听流程

1. 用户在首页或搜索结果中找到歌曲
2. 点击播放按钮
3. 系统启动播放器，开始播放歌曲
4. 播放器显示歌曲信息和专辑封面
5. 同步显示歌词，逐句滚动并高亮当前句
6. 用户可以调整音量、播放模式，或跳转到详细物料页面

### 3.5 权限管理流程

1. 超级管理员或管理员进入管理后台
2. 点击"权限管理"模块
3. 选择用户或组织
4. 设置或编辑权限
5. 可以选择预设权限模板，或自定义权限
6. 保存权限设置
7. 系统更新用户或组织的权限信息

## 4. 用户界面设计

### 4.1 设计风格

- **前端页面**：流媒体布局，类似音乐网站的界面风格，注重视觉体验和交互
- **管理后台**：经典后台布局，注重功能完整性和操作效率
- **配色方案**：主色调采用深蓝色，辅助色为白色和浅灰色，营造专业、现代的感觉
- **字体**：使用无衬线字体，确保在不同设备上的可读性
- **图标**：使用简洁、现代的图标，增强用户体验

### 4.2 响应式设计

- **设备支持**：
  - 移动端（320px-768px）
  - 平板端（768px-1024px）
  - 桌面端（1024px-1440px）
  - 大屏端（1440px以上）

- **响应式策略**：
  - 移动优先设计，确保在小屏幕设备上的良好体验
  - 使用弹性布局和媒体查询，适应不同屏幕尺寸
  - 断点优化，在关键屏幕尺寸上调整布局
  - 移动端侧边栏采用抽屉式设计，节省空间
  - 大屏端内容居中展示，提高视觉舒适度

### 4.3 页面设计概览

#### 4.3.1 登录注册页

- **布局**：居中卡片式布局
- **响应式**：
  - 移动端：全屏卡片，自适应屏幕
  - 桌面端：居中卡片，背景装饰，增强视觉效果
- **功能**：
  - 登录表单：账号、密码输入，记住密码，登录按钮
  - 注册表单：用户名、密码、邮箱等基本信息输入
  - 忘记密码：密码重置功能
  - 第三方登录（可选）：微信、QQ等第三方登录方式

#### 4.3.2 首页

- **布局**：顶部导航 + 轮播图 + 推荐列表 + 底部播放栏
- **响应式**：
  - 移动端：单列布局，轮播图高度自适应
  - 平板端：双列布局，推荐列表采用两列网格
  - 桌面端：多列网格布局，推荐内容更加丰富
- **内容**：
  - 推荐歌曲：展示系统推荐的歌曲项目
  - 最新上传：展示最近上传的物料
  - 热门榜单：展示热门歌曲榜单
  - 分类导航：按标签或分类导航到不同内容

#### 4.3.3 搜索页面

- **布局**：顶部搜索框 + 筛选栏 + 结果列表
- **响应式**：
  - 移动端：筛选栏折叠，点击展开
  - 桌面端：筛选栏展开，提供更多筛选选项
- **功能**：
  - 关键词搜索：支持模糊搜索
  - 高级筛选：多条件组合筛选
  - 标签筛选：按标签快速筛选
  - 排序选项：相关度、创建时间、发行时间、热度等

#### 4.3.4 播放器页面

- **布局**：全屏播放器 + 歌词显示 + 播放列表
- **响应式**：
  - 移动端：垂直布局，歌词和播放控制垂直排列

- **功能**：
  - 底部固定播放栏：基本播放控制，进度条，时间显示
  - 全屏播放器：专辑封面，歌曲信息，播放控制，音质选择

#### 4.3.5 物料管理页

- **布局**：顶部工具栏 + 物料列表
- **响应式**：
  - 移动端：列表视图，单列显示
  - 平板端：网格视图（2列），缩略图预览
  - 桌面端：网格视图（3-4列），更多物料同时展示

- **响应式**：
  - 移动端：侧边栏抽屉式，节省空间
  - 桌面端：侧边栏固定，提供完整导航
- **功能**：
  - 菜单折叠/展开：适应不同屏幕尺寸
  - 面包屑导航：显示当前位置
  - 快速搜索：快速查找系统内容
  - 数据导出：导出系统数据

## 5. 技术实现方案

### 5.1 技术栈

| 类别 | 技术 | 版本 | 选型理由 |
|-----|-----|-----|--------|
| 后端语言 | Java | 11+ | 成熟稳定，适合企业级应用开发 |
| 前端框架 | Vue | 3.x | 响应式设计，组件化开发，性能优秀 |
| UI组件库 | Ant Design Vue | 3.x | 丰富的组件，美观的界面，易于使用 |
| 开发语言 | TypeScript | 4.x+ | 类型安全，提高代码质量和可维护性 |
| 构建工具 | Vite | 4.x+ | 快速的开发服务器，优化的构建过程 |
| 代码规范 | ESLint + Prettier | 最新版 | 统一代码风格，提高代码质量 |
| 版本控制 | Git | 最新版 | 分布式版本控制，便于团队协作 |
| API设计 | RESTful | - | 标准化的API设计，易于集成 |
| 鉴权方案 | JWT | - | 无状态认证，便于水平扩展 |
| 数据库 | MySQL | 8.0+ | 成熟稳定，适合关系型数据存储 |

### 5.2 系统架构

- **前端架构**：
  - 基于Vue 3的单页应用
  - 使用Ant Design Vue组件库
  - 响应式设计，支持多设备
  - 模块化开发，便于维护和扩展

- **后端架构**：
  - 基于Java的RESTful API
  - 分层架构：控制器层、服务层、数据访问层
  - 权限控制：基于JWT的鉴权系统
  - 数据存储：MySQL数据库

- **数据库设计**：
  - 核心表：用户表、组织表、歌曲项目表、物料表、标签表、权限表等
  - 关系表：用户-组织关系表、项目-物料关系表、项目-标签关系表、用户-权限关系表等
  - 索引设计：为搜索字段和外键字段建立索引，提高查询性能

### 5.3 核心功能实现

#### 5.3.1 搜索和标签系统

- **实现方案**：
  - 使用MySQL全文索引或Elasticsearch实现全文搜索
  - 标签系统采用多级分类结构
  - 搜索结果排序基于相关度、时间等因素
  - 高级搜索支持多条件组合查询

#### 5.3.2 音乐预览和试听功能

- **实现方案**：
  - 使用HTML5 Audio API实现音频播放
  - 歌词同步显示通过时间轴匹配实现
  - 播放器界面采用响应式设计，适应不同设备
  - 音质选择通过不同码率的音频文件实现

#### 5.3.3 鉴权和权限管理

- **实现方案**：
  - 基于JWT的认证系统
  - 基于角色的权限控制（RBAC）
  - 权限粒度细到系统级、组织级、项目级、物料级、功能级
  - 权限模板支持预设和自定义

#### 5.3.4 物料上传管理

- **实现方案**：
  - 使用分片上传实现大文件上传
  - 支持拖拽上传和文件夹上传
  - 上传进度实时显示
  - 自动分类和标签基于文件类型和内容分析

#### 5.3.5 看板系统

- **实现方案**：
  - 使用ECharts实现数据可视化
  - 支持折线图、柱状图、饼图、雷达图等多种图表类型
  - 看板配置支持自定义和预设
  - 数据导出支持Excel、PDF等格式
