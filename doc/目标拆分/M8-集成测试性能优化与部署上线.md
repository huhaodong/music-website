# M8: 集成测试、性能优化与部署上线

> **里程碑编号**: M8
> **预计周期**: 第15周后半-16周（5-7个工作日）
> **前置依赖**: M7 全部验收通过
> **核心目标**: 全量集成测试、性能测试与优化、安全测试、生产环境部署上线

## 1. 目标概述

M8 是项目交付前的最后阶段，聚焦于质量保障和生产准备：
1. **全量集成测试**：跨模块端到端测试，覆盖全部核心业务流程
2. **性能测试与优化**：API压测、数据库查询优化、前端性能优化
3. **安全测试**：SQL注入/XSS/CSRF/越权等安全扫描
4. **生产部署**：Docker Compose 生产配置、数据迁移、上线发布

**关键约束：本里程碑不新增功能，仅修复测试中发现的问题和进行优化。**

## 3. 集成测试用例规划

### 3.1 核心业务全链路测试

```
test/integration/
├── AuthFlowIntegrationTest.java
│   ├── 完整登录流程：注册→登录→获取Token→访问受保护资源→刷新Token→登出
│   ├── 多角色登录：管理员登录→普通用户登录→各自权限验证
│   └── Token生命周期：获取→使用→过期→刷新→登出→黑名单
├── ProjectLifecycleIntegrationTest.java
│   ├── 项目全生命周期：创建→编辑→关联艺术家→上传物料→打标签→发行→暂缓
│   ├── 项目跨用户协作：用户A创建→用户B(同组织)查看→用户C(不同组织)无法查看
│   └── 项目搜索与筛选：创建多个项目→各种条件搜索→验证结果
├── MaterialFlowIntegrationTest.java
│   ├── 物料全流程：上传→预览→编辑信息→打标签→删除
│   ├── 批量上传流程：批量上传→检查状态→逐个验证
│   └── 分片上传流程：初始化→上传分片→合并→验证完整性
├── PermissionIntegrationTest.java
│   ├── 超级管理员：可访问所有资源、所有操作
│   ├── 组织管理员：仅可访问本组织及下级资源
│   ├── 普通用户：仅可访问个人相关资源
│   ├── 无权限用户：访问受控资源返回403
│   ├── 越权测试：低权限用户尝试高权限操作
│   └── 数据隔离：不同组织数据完全隔离
├── SearchIntegrationTest.java
│   ├── 全文搜索：插入测试数据→各种关键词搜索→验证结果
│   ├── 高级搜索：多条件组合→验证交集
│   └── 搜索权限：不同角色搜索同一关键词→结果应不同
└── DashboardIntegrationTest.java
    ├── 统计数据准确性：插入已知数据→统计→验证结果
    └── 看板CRUD：创建→配置→查看→更新→删除
```

## 5. 性能优化检查清单

### 5.1 后端优化

| 优化项 | 方法 | 验证 |
|--------|------|------|
| 数据库索引 | 为高频查询字段建立复合索引 | EXPLAIN 无全表扫描 |
| N+1 查询 | 使用 MyBatis Plus 的批量查询替代循环查询 | 慢查询日志 |
| 热点缓存 | Redis 缓存系统配置/权限数据/热门搜索 | 缓存命中率 |
| 连接池 | 调优 HikariCP 连接池参数 | 高并发下无连接超时 |
| 文件存储 | MinIO 预签名 URL 直传（减少后端中转） | 上传性能提升 |

### 5.2 前端优化

| 优化项 | 方法 | 验证 |
|--------|------|------|
| 代码分割 | 路由懒加载 + 组件异步加载 | 首屏 JS 体积减少 |
| 图片优化 | WebP 格式 + 懒加载 + 响应式图片 | Lighthouse 图片评分 |
| 资源压缩 | Gzip/Brotli 压缩 | 传输体积检查 |
| 缓存策略 | 静态资源长缓存 + 文件名哈希 | 缓存命中验证 |
| 骨架屏 | 关键页面添加骨架屏 | 感知加载速度 |
