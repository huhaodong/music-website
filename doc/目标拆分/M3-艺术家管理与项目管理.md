# M3: 艺术家管理与项目管理（核心业务-上）

> **里程碑编号**: M3
> **预计周期**: 第6-7周（10个工作日）
> **前置依赖**: M2 全部验收通过
> **核心目标**: 完成 Singer→Artist 概念重构、歌曲项目 CRUD 与状态机管理
> **开发分支**: `feature/project-mgmt`

---

## 1. 目标概述

M3 实现曲库管理系统的两个核心业务实体：
1. **艺术家管理**：将原有的 Singer（歌手）概念扩展为 Artist（艺术家），支持词作者/曲作者/歌手/制作人等多种角色
2. **项目管理**：新建 Project（歌曲项目）模型，替代原有 SongList（歌单），实现项目 CRUD 和状态机管理

**关键约束：原有 Singer 数据通过迁移脚本自动转为 Artist，原有歌曲数据不丢失。**

### 3.2 项目状态机设计

```
                    ┌─────────────────┐
                    │                 │
     ┌──────────────▼───┐        ┌───┴────────────┐
     │    未发行(DRAFT)  │◄───────│ 暂缓发行(HOLD) │
     └──────┬───────────┘        └───▲────────────┘
            │                        │
            │                        │
            ▼                        │
     ┌──────────────────┐           │
     │ 已发行(RELEASED)  ├───────────┘
     └──────────────────┘

合法状态转换：
  DRAFT → RELEASED    (发行)
  DRAFT → HOLD        (暂缓)
  RELEASED → HOLD     (暂缓发行)
  HOLD → DRAFT        (恢复为未发行)
  HOLD → RELEASED     (重新发行)

非法状态转换：
  RELEASED → DRAFT    (已发行不可回退为草稿)
```
