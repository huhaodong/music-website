### 2.4 分层架构说明

#### 2.4.1 Controller 层

负责接收 HTTP 请求，调用 Service 层处理业务逻辑，返回统一响应格式。

**主要控制器及其功能**：

| 控制器 | API 前缀 | 主要功能 |
|--------|----------|----------|
| AdminController | /admin | 管理员登录验证 |
| ConsumerController | /user | 用户注册、登录、信息管理 |
| SingerController | /singer | 歌手增删改查 |
| SongController | /song | 歌曲增删改查、文件上传 |
| SongListController | /songList | 歌单增删改查 |
| ListSongController | /listSong | 歌单歌曲关联管理 |
| CollectController | /collection | 收藏管理 |
| CommentController | /comment | 评论管理、点赞 |
| RankListController | /rankList | 评分管理 |
| BannerController | /banner | 轮播图查询 |
| UserSupportController | /userSupport | 点赞记录管理 |
| MinioUploadController | - | MinIO 文件上传 |

#### 2.4.2 Service 层

负责业务逻辑处理，调用 Mapper 层进行数据操作，采用接口+实现类的设计模式。

#### 2.4.3 Mapper 层

使用 MyBatis Plus 注解方式定义数据库操作，配合 XML 映射文件实现复杂查询。

#### 2.4.4 Model 层

- **domain 目录**：实体类，对应数据库表结构
- **request 目录**：请求 DTO，用于接收前端参数

### 2.5 统一响应封装

```java
public class R {
    private int code;        // 状态码（200=成功，400=错误，401=未授权，500=服务器错误）
    private String msg;      // 消息
    private Object data;     // 数据
    private boolean success; // 是否成功

    // 静态方法
    public static R success(String msg)
    public static R success(String msg, Object data)
    public static R error(String msg)
    public static R fatal(String msg)
    public static R warning(String msg)
}
```

## 4. 数据库设计概要

### 4.1 数据库信息

- **数据库名称**：tp_music
- **数据库类型**：MySQL 5.7+
- **字符集**：UTF-8
- **存储引擎**：InnoDB

### 4.2 主要数据表

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| admin | 管理员表 | id, name, password |
| consumer | 用户表 | id, username, password, sex, phone_num, email, avatar |
| singer | 歌手表 | id, name, sex, pic, birth, location, introduction |
| song | 歌曲表 | id, singer_id, name, introduction, pic, lyric, url |
| song_list | 歌单表 | id, title, pic, style, introduction |
| list_song | 歌单歌曲关联表 | id, song_id, song_list_id |
| collect | 收藏表 | id, user_id, type, song_id, song_list_id |
| comment | 评论表 | id, user_id, song_id, song_list_id, content, type, up |
| rank_list | 评分表 | id, song_list_id, consumer_id, score |
| banner | 轮播图表 | id, pic |
| user_support | 用户点赞表 | id, comment_id, user_id |

### 4.3 表关系说明

**一对多关系**：
- singer → song（一个歌手有多首歌曲）
- consumer → collect（一个用户有多个收藏）
- consumer → comment（一个用户有多条评论）
- song_list → list_song（一个歌单有多首歌曲）
- song_list → comment（一个歌单有多条评论）

**多对多关系**：
- song ↔ song_list 通过 list_song 表关联
- consumer ↔ comment 通过 user_support 表实现点赞

### 4.4 核心业务逻辑

1. **收藏逻辑**：通过 type 字段区分（0=歌曲，1=歌单）
2. **评论逻辑**：通过 type 字段区分（0=歌曲评论，1=歌单评论）
3. **评分逻辑**：用户对歌单评分唯一（consumer_id + song_list_id 联合唯一）
4. **点赞逻辑**：通过 user_support 表防止重复点赞

## 6. 启动方式

### 6.1 后端启动

```bash
cd music-server
mvn spring-boot:run
# 或使用 Maven Wrapper
./mvnw spring-boot:run
```

### 6.2 前台客户端启动

```bash
cd music-client
npm install
npm run serve
```

### 6.3 后台管理端启动

```bash
cd music-manage
npm install
npm run serve
```

### 6.4 Docker 部署

```bash
# 后端
cd docker-server
docker-compose up -d

# 前端客户端
cd docker-vue-client
docker build -t music-client .

# 管理端
cd docker-vue-manage
docker build -t music-manage .
```
