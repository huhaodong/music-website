# 权限系统重构 Spec

## Why

当前权限管理基于树形结构，业务逻辑复杂，用户操作繁琐。需要简化为基于原子权限的列表管理，管理员在角色中直接勾选权限。

## What Changes

### 核心变更

1. **权限结构简化** - 从树形结构改为扁平列表，每个权限都是独立的原子权限
2. **权限定义标准化** - 权限以功能模块 + 操作类型命名，如 `song:download`、`singer:view`
3. **角色权限关联** - 管理员在角色管理中勾选该角色拥有的权限
4. **移除权限模板** - 不再需要模板快速创建

## Impact

- Affected specs: M2-RBAC权限系统与组织用户管理
- Affected code:

  - 数据库 `permission` 表 - 重新设计

## MODIFIED Requirements

### Requirement: 权限列表管理
系统 SHALL 提供扁平化的权限列表，不再使用树形结构

#### Scenario: 查看权限列表
- **WHEN** 用户进入权限管理页面
- **THEN** 显示所有原子权限的列表，按模块分组

### Requirement: 角色权限分配
系统 SHALL 支持在角色管理中直接勾选权限

#### Scenario: 分配权限
- **WHEN** 管理员编辑角色
- **THEN** 显示权限列表，管理员勾选后保存
